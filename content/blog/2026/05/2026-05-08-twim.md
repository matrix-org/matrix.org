### Website & Content Working Group ([website](/foundation/working-groups/#website-content))

The Website and Content Working Group is responsible for the editorial and technical oversight of the main Matrix websites and social media channels.

[HarHarLinks](https://matrix.to/#/@kim:sosnowkadub.de) announces

> Recently I got up to some Rust hacking which lead us to deploy 2 updates to TWIMbot this week:
>
> 1. We implemented a [change](https://github.com/HarHarLinks/hebbot/pull/3/files) to how mentions work to reduce accidental triggering just because you start a message with "TWIMbot" or "TWIMday". This shouldn't break any existing workflows, unless I supposed you wanted to write a meta-TWIM like that to show off. I guess I should have TWIMmed before deploying. 🤔 Shout-out to Agatha for the support!
> 2. As you know we publish the "Dept of Ping" in almost every TWIM. This was an annoying process for editors, because it requires you to install and run another tool on your computer to fetch the scores and render it as a Markdown table. Unfortunately it didn't even want to run on some editing machines, and even [running it (manually) in CI](https://github.com/HarHarLinks/twim-o-matic/commit/7ba85541ad00fa7e999ac12afaf3487346d4da20) was too annoying. So we now [rewrote it in Rust](https://github.com/HarHarLinks/hebbot/pull/4)™️ and integrated it directly in TWIMbot. That also meant we had to add some error handling! One step less, and I am sure there will be noticeable reduction in editing time already in this edition.
>
> All this has already been deployed, so please excuse any possible inconvenience should it suddenly catch fire.
>
> If you're looking for a Rust project and/or improving TWIMbot, please check out the [issue tracker](https://github.com/haecker-felix/hebbot/issues/)!

### Tuwunel ([website](https://github.com/matrix-construct/tuwunel))

Enterprise successor to conduwuit, the high-performance and feature-rich fork of Conduit.

[jason 🎔](https://matrix.to/#/@jason:tuwunel.me) announces

> [Tuwunel](https://github.com/matrix-construct/tuwunel) opens a multi-release spec-compliance campaign with a [big list](https://matrix-construct.github.io/tuwunel/development/compliance/msc.html) tracking convergence to Matrix 1.18 (or 1.19). This installment lands nearly twenty MSCs and three headline features.
>
> Marquee additions: [policy server support](https://github.com/matrix-org/matrix-spec-proposals/pull/4284), gated by `enable_policy_servers` and `policy_server_request_timeout`; an [account locking and suspension](https://github.com/matrix-org/matrix-spec-proposals/pull/3939) toolkit spanning [MSC4323](https://github.com/matrix-org/matrix-spec-proposals/pull/4323) and [MSC3823](https://github.com/matrix-org/matrix-spec-proposals/pull/3823) with a new admin command that mass-rejects pending invites; and [Olm fallback keys](https://github.com/matrix-org/matrix-spec-proposals/pull/2732) re-issued on every claim so encrypted rooms survive one-time key exhaustion.
>
> Compliance batch beneath: [MSC4380](https://github.com/matrix-org/matrix-spec-proposals/pull/4380) invite blocking, [MSC4406](https://github.com/matrix-org/matrix-spec-proposals/pull/4406) `sender_ignored`, [MSC4383](https://github.com/matrix-org/matrix-spec-proposals/pull/4383) `/versions`, [MSC4260](https://github.com/matrix-org/matrix-spec-proposals/pull/4260) user reports, [MSC4373](https://github.com/matrix-org/matrix-spec-proposals/pull/4373) incoming-EDU types, [MSC4168](https://github.com/matrix-org/matrix-spec-proposals/pull/4168) space state on upgrade, [MSC4169](https://github.com/matrix-org/matrix-spec-proposals/pull/4169) redaction compat, [MSC3905](https://github.com/matrix-org/matrix-spec-proposals/pull/3905) appservice local-only namespaces, [MSC4025](https://github.com/matrix-org/matrix-spec-proposals/pull/4025) partial deactivate erase, [MSC3391](https://github.com/matrix-org/matrix-spec-proposals/pull/3391) account-data deletion, [MSC4361](https://github.com/matrix-org/matrix-spec-proposals/pull/4361) non-federating member auth, [MSC4190](https://github.com/matrix-org/matrix-spec-proposals/pull/4190) appservice cross-signing UIA bypass, [MSC4254](https://github.com/matrix-org/matrix-spec-proposals/pull/4254) OIDC revoke, and [MSC4175](https://github.com/matrix-org/matrix-spec-proposals/pull/4175) timezone keys promoted. DBendit's [request](https://github.com/matrix-construct/tuwunel/issues/316) for a public list of supported MSCs is now answered.
>
> State resolution gets four long-standing corrections (knock auth in v7-9, auth-difference, mainline 0, and v12 bootstrap-join), with [a slice upstreamed to Ruma](https://github.com/ruma/ruma/pull/2480). A `/sync` concurrency heisenbug born in Conduit, amplified by v1.3.0 optimizations, is finally zapped.
>
> lhjt [fixed](https://github.com/matrix-construct/tuwunel/pull/449) sliding-sync `bump_stamp` so Element X moves rooms even when the bumping event isn't in the response. humemm [reported](https://github.com/matrix-construct/tuwunel/issues/448) OAuth login breaking when a provider returned `expires_at` as a Unix timestamp; the upstream token DTO is decoupled. maxrdz [corrected](https://github.com/matrix-construct/tuwunel/pull/446) the example NGINX root-domain delegation. dasha-uwu landed cleanups across appservice filtering, admin lookups, room naming, and media handling.
>
> Full release notes: <https://github.com/matrix-construct/tuwunel/blob/345446255/RELEASE.md>

### Komai ([website](https://github.com/etkecc/komai))

[Slavi [etke.cc]](https://matrix.to/#/@slavi:etke.cc) reports

> We're excited to [introduce Komai](https://etke.cc/blog/introducing-komai) - a desktop-first, native Matrix chat app we've been building. Forked from [nheko](https://github.com/Nheko-Reborn/nheko) and rebuilt around [matrix-rust-sdk](https://github.com/matrix-org/matrix-rust-sdk), it's [etke.cc](https://etke.cc/)'s humble attempt at making Matrix on the desktop a little better.
>
> A few highlights:
>
> - 📑 **[Browser-style room tabs](https://github.com/etkecc/komai/blob/main/docs/user-guide/features/tabs.md)**: open multiple conversations side by side, pin the ones you keep coming back to
> - 🧭 **[A polished room directory](https://github.com/etkecc/komai/blob/main/docs/user-guide/features/room-directory.md)** with first-class [Matrix Rooms Search](https://matrixrooms.info/) support
> - 🎨 **[10+ built-in themes](https://github.com/etkecc/komai/blob/main/docs/user-guide/features/themes.md)**, all maintained to meet [WCAG AA contrast](https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html), plus [user themes](https://github.com/etkecc/komai/blob/main/docs/user-guide/features/themes.md#-user-themes)
> - 🎙️ **[Voice transcription](https://github.com/etkecc/komai/blob/main/docs/user-guide/features/voice-transcription.md)**: long-press `Space` to dictate into the composer
> - 🤖 **[Automation hooks](https://github.com/etkecc/komai/blob/main/docs/user-guide/features/automations/README.md)**: drive Komai from [CLI](https://github.com/etkecc/komai/blob/main/docs/user-guide/features/automations/cli.md), [D-Bus](https://github.com/etkecc/komai/blob/main/docs/user-guide/features/automations/dbus.md), or an AI agent over [MCP](https://github.com/etkecc/komai/blob/main/docs/user-guide/features/automations/mcp.md)
> - 👥 **[Multi-account](https://github.com/etkecc/komai/blob/main/docs/user-guide/features/application-profiles.md)** via dedicated application profiles
> - 🌐 **[30+ fully translated languages](https://github.com/etkecc/komai/blob/main/docs/maintainers/translations.md)**
>
> Komai is **Linux-only** (`x86_64`) for now, available as AppImage, Flatpak, Snap, and [on the Arch Linux AUR](https://aur.archlinux.org/packages/komai). It's fully [Free Software](https://www.gnu.org/philosophy/free-sw.html) (GPL-3.0-or-later, no CLAs, no contributor gatekeeping), [built openly with AI assistance](https://github.com/etkecc/komai/blob/main/docs/user-guide/ai.md), and good enough to ship. We're already daily-driving it ourselves. 🦁
>
> 📖 [**Read the full announcement on our blog**](https://etke.cc/blog/introducing-komai/) or head over to [the code repository](https://github.com/etkecc/komai).

### terraform-provider-matrix

[raspbeguy](https://matrix.to/#/@raspbeguy:gugod.fr) announces

> v0.3.0 is out, and it covers the _account_ side of Matrix for the first time. Prompted by feedback during the v0.2 cycle, someone asked whether the provider would ever manage profiles for bot or company accounts. Two new resources for that. 🙂
>
> The headline is **`matrix_user_profile`**, which manages the caller's global display name and avatar. Declare displayname and avatar_url in HCL, run `terraform apply`, done.
>
> Companion resource: **`matrix_user_profile_override`**, which sets a per-room displayname/avatar by editing the `m.room.member` event for a (room, user) pair. Useful for bots that want to identify differently in different rooms, for example "PagerDuty Bot" in #oncall and "StatusBot" in #general.
>
> Try it on a disposable account: feedback and bug reports very welcome. 🐛
>

### Matrix Status Monitor

[Miklós Zsitva](https://matrix.to/#/@nurefexc:nurefexc.com) reports

> Matrix Status Monitor v7 brings smarter room sorting and richer quick actions for Matrix users on GNOME.
>
> Matrix Status Monitor v7 is out, and this release makes the extension more useful as a fast Matrix navigation layer inside GNOME Shell. The update is centered around better prioritization, clearer room grouping, and faster access to your account and room actions.
>
> Room ordering now uses a weight-based multi-criteria system instead of simple timestamp sorting. That means highlights, unread counts, DMs, favourites, visit frequency, and recent activity all contribute to what you see first, which makes the panel much better at surfacing the conversations that matter right now.
>
> The extension also now separates active and idle rooms visually, which makes the room list easier to scan at a glance. In addition, menu rebuilds are scheduled asynchronously to avoid UI blocking, so the panel stays snappy even when the room state changes quickly.
>
> Notifications were upgraded too: Matrix events now produce GNOME desktop notifications with event ID deduplication, reducing noise and making alerts more trustworthy. The profile header also grew into a more useful quick-action area, with avatar display, user ID copy, and QR toggling built directly into the menu.
>
> For Matrix users who live in GNOME, v7 is about reducing friction: better prioritization, faster room access, and less repetition. It keeps the extension small and focused, while making it much more practical for everyday use.
>
> <https://extensions.gnome.org/extension/9328/matrix-status-monitor/>
