## Dept of Servers 🏢
  
### Tuwunel ([website](https://github.com/matrix-construct/tuwunel))

Enterprise successor to conduwuit, the high-performance and feature-rich fork of Conduit.

[june ✿ 🏳️‍⚧️ (it/she/puppy)](https://matrix.to/#/@june:woof.gay) reports

> Tuwunel💕 1.7.1 rebuilds federation event fetching
> 
> [Tuwunel](https://github.com/matrix-construct/tuwunel) 1.7.1 reworks how it tracks down events missing from other servers. A [new data-fetching service](https://github.com/matrix-construct/tuwunel/commit/142c1b100) runs lookups concurrently, ranks servers by membership popularity and recent reachability, and skips a request already in flight. Resolved outlier state, fetch and backoff decisions, and per-server reachability now survive a restart, so it stops re-asking the same uncacheable questions.
> 
> An [OIDC device authorization grant](https://github.com/matrix-construct/tuwunel/commit/298111654) ([RFC 8628](https://datatracker.ietf.org/doc/html/rfc8628)) lets a console or smart TV without a keyboard hand its login to a browser elsewhere. The [MSC4191](https://github.com/matrix-org/matrix-spec-proposals/pull/4191) account-management actions are complete with a deep-linked cross-signing reset, [MSC3861](https://github.com/matrix-org/matrix-spec-proposals/pull/3861) OAuth 2.0/OIDC is advertised on `/versions`, dynamic client registration is opt-in and validated, and device-scope binding now requires PKCE.
> 
> One fix comes with an apology. Since 1.6.1, the `ip_source` change had been turning away bridges and other loopback clients; that is [now fixed](https://github.com/matrix-construct/tuwunel/commit/4d4ea0cd7): they again bypass the configured `ip_source`, Unix socket included. Federation lookups recover better too: the room-summary fallback [tries every `via` server](https://github.com/matrix-construct/tuwunel/commit/5c9998374), a failed remote `publicRooms` returns a `502`, and thanks to June's [report](https://github.com/matrix-construct/tuwunel/issues/466), a bad `unix_socket_path` is named in the startup error.
> 
> June also gave `/.well-known/matrix/support` [PGP](https://github.com/matrix-org/matrix-spec-proposals/pull/4439) and [policy](https://github.com/matrix-org/matrix-spec-proposals/pull/4266) fields. Smaller spec work also lands: [MSC3980](https://github.com/matrix-org/matrix-spec-proposals/pull/3980), [MSC3860](https://github.com/matrix-org/matrix-spec-proposals/pull/3860), [MSC4311](https://github.com/matrix-org/matrix-spec-proposals/pull/4311), [MSC2659](https://github.com/matrix-org/matrix-spec-proposals/pull/2659), and a stable [MSC4267](https://github.com/matrix-org/matrix-spec-proposals/pull/4267) forget-on-leave capability, all tracked in the [compliance list](https://github.com/matrix-construct/tuwunel/blob/74fa5355b831d21f497251976d0436fd3c293e7a/docs/development/compliance/msc.md).
> 
> The rest is community work. winyadepla's [request](https://github.com/matrix-construct/tuwunel/issues/472) to purge every room containing a user is now an admin command; dasha-uwu added an `admin media preview` debug command and fixed a setting that could switch off client-side decompression; BVollmerhaus [documented](https://github.com/matrix-construct/tuwunel/pull/468) how a proxy can slip into `ip_source_trusted_subnets`. Devices can now hold several access tokens, and media fetches respect CIDR denylists.
> 
> Full release notes: <https://github.com/matrix-construct/tuwunel/blob/74fa5355b831d21f497251976d0436fd3c293e7a/RELEASE.md>
