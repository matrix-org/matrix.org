### Relay ([website](https://github.com/subpop/Relay))

[Link Dupont](https://matrix.to/#/@subpop:matrix.org) says

> Relay is a new native macOS Matrix client built with SwiftUI, wrapping the [Matrix Rust SDK](https://github.com/matrix-org/matrix-rust-sdk). The goal is a Matrix client that feels like it belongs on your Mac — fast, familiar, and following macOS design conventions.
>
> ![A screenshot of Relay, a Matrix client for macOS. It sticks to macOS Human Interface Guidelines.](/blog/img/2026-04-10-relay.avif)
> 
> Version **0.2.1** is now available. Here's what you can do with it today:
> 
> - **Rich timeline** — text, image, video, audio, and file messages with sender avatars, date headers, message grouping, and big emoji for emoji-only messages.
> - **Rich text rendering** with formatted bodies and clickable, interactive mentions.
> - **Message editing and deletion.**
> - **Reactions** via context menu, emoji picker, or long-press.
> - **Replies** — swipe on a message to reply; replies render as overlapping bubbles with click-to-jump.
> - **Pin messages** — pin/unpin messages and jump to them in the timeline.
> - **URL previews** displayed as standalone cards alongside messages.
> - **GIF search** powered by GIPHY.
> - **Username autocomplete** with inline suggestions while composing.
> - **Attachments** — stage images and files before sending via drag-and-drop, paste, or the attach button.
> - **Room directory and room creation** — discover public rooms or create new ones.
> - **Direct messages** — start a DM from any user's profile.
> - **Deep link handling** for `matrix:` URIs and `matrix.to` links.
> - **Unread markers** with a "New" divider at the first unread message.
> - **Typing indicators**, and **infinite scrollback**.
> - **Session verification** using in-app emoji verification flow.
> - **Keychain-backed sessions** for secure credential storage.
> 
> Relay targets macOS 26 (Tahoe). It's licensed under Apache 2.0.
> 
> Come say hello in [#relayapp:matrix.org](https://matrix.to/#/#relayapp:matrix.org)!

## Dept of SDKs and Frameworks 🧰

### pygomx ([website](https://codeberg.org/saces/pygomx))

[saces](https://matrix.to/#/@saces:c-base.org) reports

> a golang matrix high level client library with c-bindings for python
> 
> I wanted a library that hides all the matrix 2.0 from me and let me build my business logic in python and do qick&dirty (matrix)helper scripts. I did not found any i wanted to use, so i build my own, but with — no, I'm not Bender🤖 — a simple text based c-api and multiclient support:
> 
> - project home: [https://codeberg.org/saces/pygomx](https://codeberg.org/saces/pygomx)
> - matrix  room: [#pygomx:matrix.org](https://matrix.to/#/#pygomx:matrix.org)
> 
> pygomx is a monorepos to keep the right versions together and consists of three parts:
> 
> - libmxclient: golang matrix client library
>     - High level client library with a simple c-api: it basicly passes (json)strings back and forth.
>     - All the Matrix 2.0 magic is hidden in the lib.
>     - Should work on \*any\* platform golang is enable to build for.
> 
> - pygomx: python binding package
>     - This package contains the python bindings module binary
>     - Should work on any platform you have a decent python and c-compiler for.
>     - more: [pygomx/README.md](https://codeberg.org/saces/pygomx)
> 
> - mxsmal: simple matrix application library
>     - (might be moved into its own repository once the api kinda stabilzes)
>     - matrix client library and tools that utilizes pygomx
>     - inspired by [Simple-Matrix-Bot-Lib](https://codeberg.org/imbev/simplematrixbotlib)
> 
> 
> ##### Prebuild Binaries
> 
> Prebuild binaries are available for:
> 
> - Linux/glibc amd64/arm64
> - Mac 14+ arm64/Mac 15+ x64
> - Windows x64
> 
> In theory, this should work on these platforms:
> 
> ```
> create and activate a venv with python 3.10+
> 
>   pip install mxsmal
> 
> for a simple function test run `mxdiscover <domain.org>`. It prints the discovered client url. 
> 
> for a more sophisticated demo run the demobot:
> run `smalsetup <matrixid>` in an empty dir.
> start the bot `demobot`.
> simple demo bot. no further configuration required
> invite this bot to your room or open a DM with it.
> 
> for just the bindings library:
>   
>   pip install pygomx
> 
> ```
> 
> Have fun exploring this :)
> 
> ---
> 
> The project is at an early stage, things are still subject to change, but the basics are done and working.  
> Let's see where it evolves.

### Extera Rich Presence Companion ([website](https://github.com/ExteraApp/RichPresenceAndroid))

[rustyraven[extera.xyz op]](https://matrix.to/#/@rustyraven:extera.xyz) says

> Extera had support for [MSC4320](https://github.com/matrix-org/matrix-spec-proposals/pull/4320) rich presences for some time, but there weren't any apps or clients sending them.
>
> ![A screenshot of Extera RPC. It shows a user profile, including what they're listening to.](/blog/img/2026-04-10-extera-rpc.avif)
> 
> I've made **Extera RPC**, an Android companion app which sends MSC4320-compatible rich presence data. It supports both media and activity RPC ("listening music" and "playing"). Media detection is implemented with NotificationListenerService - any app that creates mini player in notification tray will be supported. App detection is implemented with UsageStatsManager - "playing" status will be sent as soon as you open an app. Allowed apps for media and activity RPC are configurable.
> 
> Extera RPC's source code is available on [GitHub](https://github.com/ExteraApp/RichPresenceAndroid), as well as its [initial release](https://github.com/ExteraApp/RichPresenceAndroid/releases/tag/1.0).
> 
> *Extera Rich Presence runs independently of Extera Next, you may use it alongside with any Matrix client*
> 
> [#extera:extera.xyz](https://matrix.to/#/#extera:extera.xyz) - [website](https://extera.xyz/) - [GitHub](https://github.com/ExteraApp)
